---
description: 根据最近的工作进展更新 PROJECT_NOTES.md 和 TODO.md
---

请作为项目文档维护者，根据最近的代码变更和 Git 提交记录，更新项目的核心文档。

### 1. 分析工作进展
首先，请回顾最近的对话历史、文件修改或运行 `git log -n 10` 来了解最近完成的任务。重点关注：
- 新增的功能特性
- 修复的 Bug
- 重构的代码
- 通过的测试用例

### 2. 更新 PROJECT_NOTES.md
在 `PROJECT_NOTES.md` 文件中执行以下更新：
1.  **添加新的进展记录**：在 "项目进展" -> "已完成的工作" 章节末尾（或按时间顺序的合适位置），添加一个新的小节。
    - 标题格式：`#### N. <任务名称> (<日期>)`
    - 内容包括：详细的任务描述、实现的关键类/方法、涉及的文件、测试验证结果。
2.  **更新当前状态**：在 "当前状态" 章节，将已完成的任务标记为 ✅。
3.  **更新代码库结构**：如果添加了新文件或目录，请更新 "代码库结构" 树状图。
4.  **记录关键决策**：如果有架构或设计上的重大决定，添加到 "关键决策记录" 章节。

### 3. 更新 TODO.md
在 `TODO.md` 文件中执行以下更新：
1.  **标记已完成任务**：检查文档中的任务列表，将所有已实现功能的复选框从 `[ ]` 改为 `[x]`。
2.  **更新版本状态**：如果有大版本的目标全部完成（如 v0.2.0），请更新相应的状态描述。
3.  **添加新任务**：如果发现新的缺失功能或待办事项，添加到合适的优先级列表中。

### 4. 提交更改
完成文件修改后，请生成一个 git commit 命令，建议使用如下格式的提交信息：
`docs: 更新项目笔记与待办清单，对应 <功能/任务名称>`
