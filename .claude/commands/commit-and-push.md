description: 提交并推送当前的更改到远程仓库，自动生成中文提交信息
---

请作为项目维护者，协助完成代码的提交和推送工作。请严格按照以下步骤执行：

### 1. 检查与暂存
1. 运行 `git status` 查看当前工作区状态。
2. 使用 `git add` 暂存所有相关的更改。
   - 如果确认所有更改都属于同一个逻辑变更，可以使用 `git add .`。
   - 否则，请分批添加文件。

### 2. 生成提交信息
根据暂存区的变更内容（运行 `git diff --cached` 查看），生成符合 [Conventional Commits](https://www.conventionalcommits.org/) 规范的**中文**提交信息。

**提交信息格式**：
```text
<类型>(<范围>): <描述>

[可选的正文：详细说明更改的原因、逻辑或影响]
```

**类型说明**：
- `feat`: 新功能
- `fix`: 修复 Bug
- `docs`: 文档变更
- `style`: 代码格式调整（不影响逻辑）
- `refactor`: 代码重构（既不是新增功能也不是修复 Bug）
- `perf`: 性能优化
- `test`: 添加或修改测试
- `chore`: 构建过程或辅助工具的变动

### 3. 执行提交与推送
1. 使用生成的提交信息执行 `git commit`。
2. 执行 `git push` 将更改推送到远程仓库。
